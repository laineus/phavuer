# Functions

## useGame()

Get current instance of [Phaser.Game](https://newdocs.phaser.io/docs/4.1.0/Phaser.Game)

It can be used in the setup of a child component of game.

```html
<script setup>
const game = useGame()
</script>
```

## useScene()

Get current Instance of [Phaser.Scene](https://newdocs.phaser.io/docs/4.1.0/Phaser.Scene)

It can be used in the setup of a child component of scene.

```html
<script setup>
const scene = useScene()
</script>
```

## onPreUpdate(event)

A method to register an event on pre update of the scene.

It can be used in the setup of a child component of scene.

```html
<script setup>
onPreUpdate((time: number, delta: number) => {
  // ..
})
</script>
```

## onPostUpdate(event)

A method to register an event on post update of the scene.

It can be used in the setup of a child component of scene.

```html
<script setup>
onPostUpdate((time: number, delta: number) => {
  // ..
})
</script>
```

## refPhaserInstance\<T\>(value)

A typed ref that holds a Phaser instance exposed from a Phavuer component via `defineExpose`.

**Parameters:**

`value`: Initial value (typically `null`)

**Return value:**

A Vue `Ref` whose value is set to the component's `phaserInstance` when the component is mounted.

**Usage:**

```js
const rectangle = refPhaserInstance<Phaser.GameObjects.Rectangle>(null)
```

```html
<Rectangle ref="rectangle" />
```

After mount, `rectangle.value` holds the underlying `Phaser.GameObjects.Rectangle` instance.

# Functions (for contributors)

## initGameObject(object, props, options?)

This method endows the specified object with the following features:

- Reactivity to specified props, such as `x` or `y` ([full list here](https://github.com/laineus/phavuer/tree/master/src/setters.ts)).
- Automatic appending to the parent Container, if it exists.
- If a parent Container does not exist, it is automatically appended to the Scene (except for FX objects and physics bodies).
- If inside a RenderTexture context, it is appended to the RenderTexture render list.
- For Light objects, it is registered to `scene.lights` (and enables the Light system if needed).
- Automatic destruction when the component is unmounted.
- Ability to set interactive events like `@pointerup`.
- Ability to set an event for object creation using `@create`.

Parameters:

- `object`: Phaser 4 GameObject / Arcade Body / FX Controller / Light instance
- `props`: Vue 3 props
- `options`: Optional config (`{ isFx?: boolean }`)

This method is utilized to define GameObject Components. ([Example here](https://github.com/laineus/phavuer/tree/master/src/components/Sprite.vue))

If you simply wish to use your component within another component, this method is not necessary.  
In such cases, you only need to relay props to the default components.
